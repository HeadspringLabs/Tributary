Buttons

Buttons can be easily styled by adding the .button class to an element. Primary actions—most pages will have only one—can be signified by adding .primary, which adds additional styling.

It's important to style as buttons only those elements that directly respond to user interactions. Styling visual elements as buttons causes confusion.

Primary Secondary

Buttons in Forms

Forms typically have two types of actions: one to submit the form, and another to either reset it, or return to a previous location. Tributary provides a way of building confidence in the former while not preventing the latter.

Reset

User Experience Considerations

Buttons are used to indicate an action may be performed (examples include saving data, performing a search, or downloading a file). Navigational actions, such as returning to a previous page, should be styled as links, not buttons.

Buttons should be placed to the bottom-right of the content section for which they are most applicable.
